# 设定 Inspector 概览卡 + 详情弹窗设计

- 日期：2026-04-19
- 主题：右侧 Inspector `设定` 面板从“完整阅读区”改为“概览卡 + 可复用详情弹窗”
- 范围：`frontend/src/components/tabs/SetupTab.vue` 及其相关展示组件、弹窗组件、测试、回归脚本
- 结论：右侧 `设定` 面板取消内部二级 tabs，改为同时展示 `角色 / 世界观 / 核心概念` 三张概览卡；完整信息迁移到一个可复用模态弹窗中，弹窗内部再切换 `角色 / 世界观 / 核心概念`

## 1. 目标与非目标

目标：

- 让右侧 `设定` 面板在有限空间内一次性展示更多关键信息，而不是一次只看一块。
- 把“概览”和“深读”明确分层：右侧只做概览，完整信息统一进可复用弹窗。
- 复用当前已经做好的完整展示能力，而不是再造一套详情渲染。
- 为后续故事线、大纲、版本等面板复用同类详情窗口预留结构。

非目标：

- 本轮不把弹窗扩展到故事线、大纲、版本等其他面板，只先为 `设定` 接入。
- 本轮不在弹窗内提供直接编辑。
- 本轮不新增后端字段，不改 API 协议。

## 2. 当前问题

当前实现的问题不是“信息不够”，而是“信息层级错误”：

- 右侧 `设定` 面板直接承载了完整阅读内容，导致它再次变重、变深、变挤。
- 用户如果只想快速扫一眼当前设定概况，成本太高。
- 右侧天生是辅助面板，不该承担完整内容消费职责。

结论：

- 右侧应该是概览面板，不是完整阅读器。
- 完整内容需要独立窗口承接。

## 3. 核心决策

采用“右侧三卡概览 + 单一复用弹窗”的方案。

结构：

- 右侧 `设定` 面板：同时展示三块概览
  - `角色`
  - `世界观`
  - `核心概念`
- 每块只展示摘要，不展示完整内容。
- 点击 `查看完整` 后，打开一个可复用模态弹窗。
- 弹窗内部再切 `角色 / 世界观 / 核心概念`，并承载完整信息。

不采用的方案：

- 不保留右侧内部二级 tabs。
  原因：一次只能看一块，不符合“右侧一次多看点信息”的目标。
- 不做右侧行内展开。
  原因：会再次把右侧撑成完整阅读区。

## 4. 信息架构

### 4.1 右侧概览卡

右侧 `设定` 面板改为 3 张概览卡，顺序固定：

1. `角色`
2. `世界观`
3. `核心概念`

每张卡都包含：

- 标题
- 1 段到 3 段精简摘要
- 可选辅助标签
- `查看完整` 操作

### 4.2 角色概览卡

角色卡不再展示完整主从视图，只给“够决策”的信息。

建议内容：

- 角色总数
- 最关键的 1 到 2 名角色
- 每名角色展示：
  - 姓名
  - 1 行摘要：优先背景，其次性格，其次目标
  - 可选元信息：年龄 / 性别 / 状态

如果没有角色：

- 显示“暂无角色设定”
- 仍保留 `查看完整`

### 4.3 世界观概览卡

世界观卡不展示 5 张完整字段卡，只提炼最关键的 2 到 3 条摘要。

建议优先级：

1. `时代背景`
2. `社会结构`
3. `规则体系`

如果某项为空：

- 跳过该项
- 全空则显示“世界观待补充”

### 4.4 核心概念概览卡

核心概念卡同样做摘要化。

建议优先级：

1. `主题`
2. `前提设定`
3. `核心钩子`

如果全空：

- 显示“核心概念待补充”

## 5. 详情弹窗

### 5.1 交互模型

使用模态弹窗，不用抽屉。

原因：

- 弹窗最适合承接完整阅读，不挤压右侧面板。
- 抽屉会和右侧 Inspector 的空间关系打架。
- 模态更利于后续复用。

打开来源：

- 点任意概览卡的 `查看完整`

打开后行为：

- 弹窗打开
- 默认定位到对应 section
  - 从 `角色` 卡进入 -> 弹窗默认 `角色`
  - 从 `世界观` 卡进入 -> 弹窗默认 `世界观`
  - 从 `核心概念` 卡进入 -> 弹窗默认 `核心概念`

关闭方式：

- 右上角关闭按钮
- 遮罩点击关闭
- `Escape`

### 5.2 弹窗内容结构

弹窗包含：

- 头部
  - 标题：如“设定详情”
  - 关闭按钮
- 内部 tabs
  - `角色`
  - `世界观`
  - `核心概念`
- 内容区

内容区复用现有完整展示能力：

- `角色`：沿用当前主从视图
- `世界观`：沿用当前 5 张字段卡
- `核心概念`：沿用当前 4 张字段卡

要求：

- 不要复制一套完整渲染逻辑
- 通过组件复用实现“同一份完整视图，既能进弹窗，也能在测试中单独验证”

## 6. 组件边界

建议拆分如下：

- `SetupTab.vue`
  - 只负责右侧概览卡壳层
  - 负责弹窗开关状态和入口 section
- `SetupSummaryCards.vue`
  - 负责三张概览卡容器
- `SetupSummaryCard.vue`
  - 负责单张概览卡
- `InspectorDetailModal.vue`
  - 通用模态弹窗壳层
- `SetupDetailModal.vue`
  - `设定` 专用详情弹窗装配层
- 现有完整组件继续保留：
  - `SetupCharactersPanel.vue`
  - `SetupWorldPanel.vue`
  - `SetupConceptPanel.vue`

约束：

- 不把概览和完整视图重新堆回 `SetupTab.vue`
- 弹窗壳层尽量通用，别做成只服务 setup 的一次性垃圾组件

## 7. 状态模型

状态分两层：

右侧概览层：

- 无需再维护 `activeSection`

弹窗层：

- `isDetailModalOpen: boolean`
- `detailModalSection: 'characters' | 'world' | 'concept'`

角色完整视图层：

- 继续沿用当前 `activeCharacterToken + snapshot` 机制
- 只存在于完整视图容器中

关键规则：

- 关闭弹窗后，不影响右侧概览卡状态
- 再次打开弹窗时，以最新点击入口 section 为准

## 8. 视觉要求

右侧概览卡要更轻、更短、更可扫读。

要求：

- 三张卡并列纵向排布，留白比完整视图更紧
- 每张卡控制摘要长度，避免再把右侧拉成长文阅读区
- `查看完整` 按钮低调但明确，不做主 CTA 大按钮
- 弹窗内容区可以更宽、更适合阅读

弹窗视觉要求：

- 沿用羊皮纸体系
- 但正文区域保持干净浅底，优先阅读
- 不走厚重管理后台风格

## 9. 测试策略

至少补这些覆盖：

- 右侧默认同时看到三张概览卡
- 点击 `角色 / 世界观 / 核心概念` 的 `查看完整`，弹窗打开且定位正确
- 弹窗内可切换三类内容
- 世界观和核心概念依旧不暴露 JSON key
- 角色区完整视图仍保持重名角色稳定选中逻辑
- 关闭弹窗后，页面主状态正常

回归脚本需补：

- 点击右侧 `设定`
- 校验三张概览卡同时存在
- 点 `查看完整`
- 校验弹窗打开并定位到目标 section

## 10. 风险与取舍

风险：

- 如果概览摘要提炼不当，右侧可能过空或过重。
- 弹窗如果做得太专用，后续复用价值会很差。

处理：

- 概览只保留“能帮助用户决定是否点开”的信息。
- 完整内容完全复用现有组件，不再复制逻辑。
- 弹窗壳层独立出来，为后续别的 Inspector 内容复用留接口。

最终取舍：

- 右侧优先“多看概况”
- 完整阅读交给弹窗
- 不再让右侧承担完整阅读职责
