# Longform Scale Phase 45: 前端分页总数口径修正

## 目标

千章项目中，前端通常只加载当前章节页。当前部分 UI 会把已加载章节数误显示为全书章节数，导致 Athena 上下文和 Hermes 总览低估项目规模。本阶段改为优先使用后端返回的章节总数和项目总字数。

## 验收标准

1. Athena 对话面板显示 `chaptersTotal`，没有总数时才回退到已加载章节数。
2. Hermes 生产总览显示后端章节总数，不把分页页长当作全书长度。
3. Hermes 字数优先使用项目 `current_word_count`，没有该字段时才回退到已加载章节求和。
4. 相关组件测试、前端测试、后端测试和构建通过。
