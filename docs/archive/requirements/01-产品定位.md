## 1. 核心定位

墨舟 AI Writer 是一款 **AI 全自动长篇写作系统**，通过单一入口的对话式交互，让 AI 自主完成从创意到成品的完整写作流程。

### 1.1 四大核心能力

| 能力 | 描述 | 关键指标 |
|------|------|----------|
| **AI全自动写作** | AI 自主完成设定 → 大纲 → 正文的完整流程 | 人工干预率 < 5% |
| **长篇一致性** | 确保角色、情节、伏笔在长篇中保持一致 | 一致性错误 < 0.1% |
| **自优化机制** | AI 根据反馈自我修正和优化 | 迭代收敛 < 3 轮 |
| **反馈系统** | 完善的用户反馈收集和处理机制 | 反馈响应 < 2 秒 |

### 1.2 设计理念

| 原则 | 说明 |
|------|------|
| **单一入口** | 对话即一切，用户通过单一对话界面完成所有操作 |
| **极简美学** | 黑白灰主调，60%+ 留白，减少认知负荷 |
| **透明可控** | 进度可视、审批保留、用户随时可查看和干预 |
| **智能闭环** | 反馈 → 学习 → 优化，形成自我改进循环 |
| **长篇保障** | 拓扑图 + 一致性引擎，保障长文本质量 |
| **持续进化** | 偏好学习、自我改进，越用越懂用户 |

### 1.3 交互模式

- **单一入口对话式**：用户通过单一对话界面完成所有操作
- **用户审批权**：所有关键节点（设定 / 大纲 / 章节）需用户审批
- **用户查看权**：用户可随时查看进度、内容、拓扑图
- **用户不直接写作**：用户只审批和提出修正，不直接写作

## 2. 系统边界

### 2.1 系统负责

- 创意生成与扩展
- 世界观和角色设定
- 故事大纲规划
- 章节正文生成
- 一致性检查和维护
- 用户意图理解
- 审批流程管理
- 根据反馈自我优化

### 2.2 系统不负责

- 最终出版格式排版
- 多语言翻译
- 外部平台发布
- 实时协作编辑
- 图片 / 音频生成

## 3. 架构设计原则

| 原则 | 说明 |
|------|------|
| **单一职责 (SRP)** | 每个模块只做一件事，做好一件事 |
| **依赖倒置 (DIP)** | 依赖抽象而非具体实现 |
| **最小依赖** | 模块间依赖关系清晰且最小化 |
| **可测试性** | 每个模块可独立测试 |
| **可扩展性** | 新功能可通过扩展点接入 |
| **故障隔离** | 局部故障不影响整体系统 |
| **可观测性** | 关键操作可追踪、可监控 |

### 3.1 渐进式架构哲学

> **"先跑起来，再跑得好"**

- **MVP 阶段**：聚焦核心写作流程（设定 → 大纲 → 单章生成），快速验证价值
- **成长阶段**：完善一致性检查、反馈系统、版本管理
- **成熟阶段**：引入多 Agent 协作、A/B 测试、插件生态
